# Design direction - pick a theme (team vote)

Hello Team. At the week-6 Wednesday class review, the lecturer asked us to make SwiftChoice more lively and add
some game feel. I have taken that feedback, plus the earlier retro direction from Tracy, and built four
interactive mockups of our whole app. The point is that we all see and feel
each theme across every screen before we commit, instead of styling screen by
screen and ending up with a mismatched app.

All four are the same app, with the same new game features (levels, XP, quests,
streaks, badges) and the same new Fuel additions (an exact-budget option and
cuisine chips). Only the visual theme changes. Whichever we pick, we build it with
the shared design tokens, so every screen matches automatically and we can restyle
from one place later.

## The four options (open each on your phone and move through the screens)

1. **Glass** - https://swiftchoice-glass.netlify.app/
   Soft light gradient with frosted, translucent cards and friendly rounded
   avatars. Calm, airy, premium.

2. **Arcade** - https://swiftchoice-arcade.netlify.app/
   A bold dark dashboard with glowing accents and a strong game feel. The most
   energetic option.

3. **Retro** - https://swiftchoice-retro.netlify.app/
   Warm cream tones, pixel-style avatars, and chunky friendly type. Playful and
   characterful. This is the original retro idea applied across the whole app.

4. **Original (refined)** - https://swiftchoice-mini-design.netlify.app/
   Our current design system look, cleaned up, with the new game features added.
   The most familiar option and the smallest change from where the app is now.

## How to vote

There is a poll in our WhatsApp group. Have a proper look on your phone first, open
all four links and move through the screens, then vote for your first choice. If
you feel strongly, add a quick note on why.

## What happens next

The winner becomes our theme. We set its colours and styles once in the design
tokens, and then each of us restyles our own screens to match, so there is one
agreed look and nobody is guessing. Your own screen logic does not change; only the
colours and styling get pointed at the chosen theme's tokens.
